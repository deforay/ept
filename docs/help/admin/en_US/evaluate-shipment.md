# View / Re-Evaluate Shipment

You reach this page by clicking **View** on a shipment in
**Evaluate PT Survey**. It shows every participant's response for
that one shipment. From here you can:

- Re-score the whole shipment.
- Edit or change a single participant's result.
- Attach a corrective action file.
- Write a comment that all participants will see.

## At the top of the page

- **You are currently evaluating** *(shipment code)* — tells you
  which shipment you are looking at.
- **Re-Evaluate** *(red button)* — starts a fresh round of scoring
  for this shipment. The page reloads once the job is queued.
  Disabled after the shipment is finalized.
- **Back** — returns to **Evaluate PT Survey**.
- A small table shows the **Scheme Type**, **PT Survey Code** and
  **Date**.

## Corrective Action File and shared comment

Just below the header there is a small form. Both fields apply to
**every** participant in this shipment and both appear in the
final reports.

### Corrective Action File

One file (usually a PDF) for participants who did not pass. It
explains what to do next.

- Click **Choose file** to pick a file from your computer.
- Only one file is kept per shipment. Uploading a new file
  replaces the old one.
- A **Corrective Action** download button shows up next to the
  participant's report, so they get this guidance with their
  result.
- To remove the file, just upload a new one in its place.

### Comment for all Participants of this shipment

A free-text note that appears on every participant's report.
Common uses:

- Explain a problem with a sample.
- Sum up how everyone did overall.
- Flag a known issue with the panel.

The text box always shows the current saved comment. Saving
replaces the old comment — there is only one comment per
shipment. If you need to keep the old text, copy it out first.

### Update File & Comment

One button saves both fields together. You can save the comment
on its own without picking a file. You can edit and save as many
times as you like, until the shipment is finalized.

## Filter to only changed results

Above the table, the **Show only manually overridden results**
dropdown lets you see only the rows you have changed by hand.
Handy when you want a final review before generating reports.

## Participants table

One row per participant, with these columns:

- **Participant/Tester** — lab name and ID.
- **Response Score** — score from the test results.
- **Documentation Score** — score from documentation questions.
- **Result** — Pass, Fail, or Excluded. Rows you have changed by
  hand are clearly marked.
- **Response Status** — whether the participant responded.
- **Responded On** — when they submitted.
- **Last Modified** — when the row was last changed.
- **Action** — per-row buttons (see below).

## Action buttons on each row

- **View** — see exactly what the participant submitted,
  read-only.
- **Edit** — open the participant's response form so you can
  correct typos or obvious entry errors on their behalf. Your
  changes are marked as a manual change.
- **Override Result** — change the Pass / Fail, or mark the
  participant as Excluded with a reason.
- **Delete Response** — clear the participant's response so they
  can re-submit. You will be asked to type to confirm — this
  protects against accidental clicks.

## Tips

- **Re-Evaluate after any change.** Editing a row or changing a
  result does not re-score the whole shipment on its own. Click
  **Re-Evaluate** so the totals on the previous page match.
- **Use the comment and corrective action file** whenever many
  participants struggled. Both appear in the PDFs they receive.
- **After finalize, this page is read-only.** Re-Evaluate is
  disabled and your edits no longer reach participants.
